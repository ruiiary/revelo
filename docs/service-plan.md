## 2. 기획 배경 및 필요성

### Why 필사? — 학습과학적 근거

필사는 단순한 베껴쓰기가 아니다. 아래 인지심리학적 메커니즘을 통해 학습 효과를 극대화한다.

- **인출 연습 효과 (Retrieval Practice Effect)**: 읽기만 하는 것보다 기억해서 꺼내 쓰는 행위가 장기 기억을 더 강하게 형성한다.
- **생성 효과 (Generation Effect)**: 직접 생성한 정보는 수동적으로 받아들인 정보보다 기억에 더 오래 남는다.
- **유창성 착각 극복 (Overcoming Fluency Illusion)**: 읽어서 이해한 것 같지만 막상 써보면 기억나지 않는 현상. 필사는 실제 언어 사용 능력과 체감 능력 사이의 간극을 메운다.

### Why Revelo? — 왜 이 서비스가 필요한가

**문제 1. 읽는 것과 쓰는 것 사이의 단절**

좋은 문장을 읽고 이해한 것 같지만 막상 써보면 기억나지 않는 유창성 착각(Fluency Illusion). 필사는 이 간극을 메우는 가장 검증된 방법이지만, 이를 체계적으로 지원하는 디지털 서비스가 없다.

**문제 2. 좋은 문장이 흩어지고 사라진다**

메모 앱·사진 갤러리·SNS에 분산 저장되어 나중에 꺼내 학습에 활용하기 어렵다.

**문제 3. 루틴이 끊긴다**

성취 시각화나 루틴 유도 장치 없이는 대부분 초반에 중단한다.

### 기존 서비스 비교

| 서비스          | 쓰기 학습 | 아카이빙 | 루틴 유도 |
| --------------- | --------- | -------- | --------- |
| Duolingo        | ✗         | ✗        | ✓         |
| 노션 / 에버노트 | ✗         | ✓        | ✗         |
| 밀리의서재      | ✗         | △        | ✗         |
| **Revelo**      | **✓**     | **✓**    | **✓**     |

### 페르소나

> **이름**: 김지원 (24세, 대학교 4학년)
>
> **상황**: SNS에서 좋은 영어 문장을 발견하면 캡처하지만 다시 꺼내보지 않는다. B2 수준에서 멈춰있는 영어 실력을 끌어올리고 싶지만, 꾸준히 이어갈 방법을 찾지 못하고 있다. 필사를 한번 시도해봤지만 며칠 만에 그만두었다.

### AS-IS → TO-BE

**AS-IS (현재)**

- 좋은 문장을 발견해도 메모앱·갤러리·SNS에 분산 저장, 다시 꺼내 쓰기 어려움
- 필사는 노트와 펜이 있어야 하고, 어디서 시작해야 할지 모름
- 내가 어떤 책에서 얼마나 필사했는지 확인할 방법이 없음
- 루틴을 혼자 유지해야 하고, 성취감을 느낄 장치가 없어 금방 그만둠
- 영어 문장을 필사해도 공부 효과가 있는건지 확인이 안 됨

**TO-BE (Revelo)**

- 읽은 책·글에서 문장을 직접 등록하거나, 서비스 추천 문장으로 바로 필사 시작
- 문장을 보여주다 자동으로 가리고 → 기억해서 타이핑 → 원문과 비교해 틀린 부분 즉시 확인
- 필사한 책들이 두께감 있는 책장 UI로 시각적으로 쌓임
- 연속 필사 일수 스트릭과 누적 문장 수로 성취감 자극
- 틀린 문장은 오답 노트에 자동 수집되어 반복 학습 가능

## 3. 핵심 기능 정의

### 기능 1 — 문장 입력 (두 가지 경로)

**A. 추천 문장 피드**

- 서비스가 큐레이션한 문장 제공
- 언어(한국어·영어 등)·장르(문학·에세이·명언 등) 필터 지원
- 문장 단위로 필사 바로 시작 가능

**B. 직접 입력**

- 내가 읽는 책·글에서 직접 passage 등록
- 입력 항목: 문장 본문, 책 제목, 저자, 페이지, 언어
- 등록된 문장은 내 책장에 해당 책 아래 분류

---

### 기능 2 — 필사하기 (핵심 UX)

**플로우: Read → Cover → Write → Compare → Retry**

```
1. Read      원문 문장 표시
2. Cover     문장 자동으로 가리기 (타이밍 사용자 선택 가능)
3. Write     타이핑으로 써보기 (감성 폰트·따옴표 스타일 선택)
4. Compare   원문과 내가 쓴 것 나란히 비교 / 틀린 부분 하이라이트
5. Retry     틀린 부분 다시 써보기 또는 전체 재시도
```

**상세 흐름**

1. 필사할 문장 선택 (직접 입력 또는 큐레이션 문장 중 선택)
2. 한 문장씩 원문 확인 후 암기하여 문장 단위로 작성
3. 원문과 비교 — 틀린 문장은 자동으로 오답 노트에 저장
4. 문장 즐겨찾기 가능 (나중에 다시 필사)

---

### 기능 3 — 내 책장 아카이브

- 직접 등록한 책들이 책장에 쌓이는 UI
- 필사 문장 수에 비례해 책 두께(시각적 볼륨) 변화
- 날짜별·언어별·책별 필터로 내 필사 기록 탐색
- **스트릭**: 연속 필사 일수 표시
- **통계**: 누적 필사 문장 수, 누적 단어 수

---

### 기능 4 — 스크랩 & 오답 노트

**스크랩 (즐겨찾기)**

- 마음에 드는 문장 즐겨찾기
- 컬렉션 구성 (직접 그룹핑)

**오답 노트**

- 필사 중 틀린 문장 자동 수집
- 오답 노트 문장으로 재필사 가능 (반복 학습)

## 5. 기술 스택

### 확정 스택

| 레이어       | 기술                                 |
| ------------ | ------------------------------------ |
| Frontend     | Next.js (App Router), TypeScript     |
| Styling      | Styled-components                    |
| Backend / DB | Supabase (Auth, PostgreSQL, Storage) |
| Auth         | 카카오 소셜 로그인 (OAuth)           |
| Deployment   | Vercel                               |

### 선택 이유

- **Next.js App Router**: 파일 기반 라우팅으로 구조가 명확하고, SSR/SSG 지원으로 초기 렌더링 최적화
- **Supabase**: Auth + DB + Storage를 하나의 플랫폼에서 관리 가능. 1인 개발에 적합한 BaaS
- **Styled-components**: 컴포넌트 단위 스타일 관리, 동적 스타일(필사 폰트 선택 등)에 유리
- **Vercel**: Next.js와 네이티브 통합, 무료 티어로 MVP 배포 가능

### Phase 2 — 문장 입력 (2~3주)

**큐레이션 피드**
- [ ] Supabase `sentences` 테이블에 초기 큐레이션 데이터 시딩 (영어 10문장 이상, 한국어 10문장 이상)
- [ ] 홈 대시보드(`/dashboard`) 화면 구현 — 스트릭, 오늘의 큐레이션 카드
- [ ] 언어 설정(로컬 스토리지)에 따라 큐레이션 필터링 적용
- [ ] 추천 문장 카드 컴포넌트 (제목, 저자, 문장 미리보기, 필사 시작 CTA)

**직접 입력**
- [ ] 책 등록 폼 (제목, 저자, 언어 선택)
- [ ] 문장 직접 입력 폼 (문장 본문, 책 연결, 페이지 번호)
- [ ] 입력한 문장 목록 조회 (로컬 스토리지 기반)

---

### Phase 3 — 필사 UX (3~4주) ← 핵심

**5단계 플로우 구현**
- [ ] **Read**: 문장 전체 표시 화면, 준비 완료 버튼
- [ ] **Cover**: 문장 가리기 전환 (fade-out 또는 blur 애니메이션)
- [ ] **Write**: 텍스트 입력 영역 (모바일 키보드 대응, 자동 포커스)
- [ ] **Compare**: 원문 vs 입력 나란히 비교, 틀린 부분 하이라이트 (diff 로직)
- [ ] **Retry**: 틀린 문장만 재시도 또는 전체 다시하기

**상태 & 기록**
- [ ] 5단계 진행 표시 UI (스텝 인디케이터)
- [ ] 필사 완료 시 로컬 스토리지에 `practice_logs` 저장 (정답 여부, 시각 포함)
- [ ] 틀린 문장 자동 오답 플래그 처리 (`is_correct: false`)

---

### Phase 4 — 아카이브 & 통계 (2주)

**내 책장**

- [ ] `/library/[bookId]` 책별 필사 문장 목록
